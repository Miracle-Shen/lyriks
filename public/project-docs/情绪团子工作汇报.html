情绪团子

情绪团子是音乐 App 内的情绪陪伴型宠物。它把用户当下的一级情绪、二级动作状态、听歌场景、推荐内容和轻量互动连接起来，让用户在听歌时获得更明确的陪伴感。

产品描述 基于《产品方案.docx》，结合 MuChator 中 active music discovery 和 conversational recommendation 的表述。
3 个主竞品 汽水音乐、QQ 音乐、网易云音乐作为重点，其余产品仅作补充参考。
MVP 验证 demo 效果：浮动团子、节奏动效、一级情绪、二级动作状态、情绪推荐入口。
商业闭环 提升推荐消费和播放停留，再承接状态皮肤套装、动作资产和联名收入。

Tab 1 · 产品描述

一个情绪陪伴型音乐宠物，承接用户“当下想听什么”的模糊需求。

产品主语言来自《产品方案.docx》：用户先认领基础团子，再设置一级情绪和可选的二级动作状态。一级情绪表达“我现在是什么感觉”，二级动作状态表达“我现在正在做什么，想以什么样子待在这个情绪里”。结合 MuChator 论文语境，情绪团子承接的是情境化、口语化、主动表达的音乐发现需求。

一句话定义

情绪团子是音乐 App 内的情绪陪伴型宠物。用户选择或创建自己的团子形象，并为团子设置“一级情绪 + 二级动作状态”。团子出现在播放页或音乐空间中，随音乐节奏、歌词氛围和情绪转折做出反应，并影响推荐歌单的内容方向。

例如用户选择“放松 + 正在旅行”，团子可以呈现旅行家皮肤，背着包、拿着地图，在海边或城市背景中移动；推荐歌单也会更偏旅行、民谣、Chill 或轻松氛围。

核心定位

  • 不是单纯养宠，而是用户听歌过程中的情绪代理。
  • 不是固定动画，而是根据音乐节奏、歌词氛围、情绪状态和用户反馈变化的音乐 Agent 雏形。
  • 不是普通推荐歌单包装，而是把推荐理由、播放反馈和个人音乐身份前台化。
认领情绪团子 用户选择基础团子形象，作为自己在音乐 App 中的陪伴宠物。
设置两层状态 先选择一级情绪，再选择可选的二级动作状态，例如放松 + 正在旅行。
情绪适配推荐 一级情绪决定推荐基调，二级动作状态细化场景歌单和推荐标签。
陪伴听歌 团子根据节奏、音量、歌词氛围和情绪转折跳跃、漂浮、慢晃或发光。
形象成长 用户通过任务、活动或成就解锁更多动作状态和皮肤套装。

主体音乐形象

团子是用户在音乐 App 虚拟空间中的基础形象，可以从圆滚滚团子扩展到动物型、精灵型、漂浮型等形态。

一级情绪表现

一级情绪决定团子的颜色、光效、表情和整体氛围，也决定推荐的大方向。

二级动作状态

动作状态决定皮肤套装、道具、动作和场景背景，也会影响更细的推荐标签。

情绪与状态体系

新版产品方案将团子状态拆成两层：一级情绪回答“我现在是什么感觉”，二级动作状态回答“我现在正在做什么，想以什么样子待在这个情绪里”。情绪保持克制，场景、曲风、语言和活动沉淀为动作状态或推荐标签。

Tab 2 · 竞品分析

主竞品聚焦汽水音乐、QQ 音乐、网易云音乐。

根据《三种app中陪伴小宠形象调研.docx》的文字描述，三组样本分别代表三种方向：汽水音乐把小宠物接到 AI 音乐发现入口，QQ 音乐把 AI 陪伴做成可选人物、动作和背景玩法，网易云音乐把播放页宠物与自定义生成结合。

汽水音乐：模式选择、小宠物与 AI 音乐发现

  • 调研文档中记录了“多场景模式选择”“浮动小宠物”“点击小宠物跳转 Assistance：AI 音乐发现”。
  • MuChator 论文对应沉浸式信息流发现和交互式音乐推荐，说明自然语言场景找歌已经被产品化验证。
  • 局限是小宠物互动性弱，更像 AI 搜歌助手快捷入口；情绪团子需要让角色本身承担情绪选择和推荐反馈。

QQ 音乐：AI 陪伴、人物选择与背景玩法

  • 调研文档中记录了 AI 陪伴人物选择、人物动作、音乐播放页固定位和背景玩法。
  • 它证明音乐产品里可以承接人物、动作和背景这类视觉资产，适合和会员、装扮体系连接。
  • 局限是音乐页形象位置固定，不可玩、不可交互、不可移动；情绪团子应强调可拖拽、可切换、随音乐变化。

网易云音乐：自定义生成与音乐人格表达

  • 调研文档中记录了播放页宠物和自定义生成宠物形象。
  • 网易云的优势在于音乐情绪、评论社区、音乐人格和状态表达心智，适合做“我的音乐状态”。
  • 局限是宠物主要在固定动作中切换；情绪团子应把生成形象、听歌行为和推荐状态连续绑定。
产品 重点能力 情绪团子的机会
汽水音乐 多场景模式选择、浮动小宠物、Assistance 入口、沉浸式推荐流、交互式音乐推荐。 保留主动找歌能力，同时让团子不仅是入口，还能表达情绪、解释推荐和承接反馈。
QQ 音乐 AI 陪伴人物选择、人物动作、音乐固定位、背景玩法、会员和装扮资产。 吸收人物、动作、背景资产能力，同时解决固定、不可玩、不可移动的问题。
网易云音乐 播放页宠物、自定义生成宠物形象、音乐情绪社区、音乐人格和状态表达。 把“生成一个宠物”升级为“宠物随听歌状态持续变化”。
其他参考 全民 K 歌可作为娱乐化角色动作参考；Spotify/Deezer 强调 mood 推荐和推荐解释。 简单借鉴，不作为本方案主竞品。重点仍回到中文音乐产品的播放、推荐、会员和社交场景。

Tab 3 · 游戏拓展

用“听歌掉落”把陪伴感变成可持续的轻互动。

小游戏采用“轻放置 + 微节奏收集 + 情绪陪伴”的形式：用户不用一直盯着屏幕玩，听歌本身会自动掉落资源；用户想互动时，可以点一点、接一接、抱一抱小宠物，帮它把情绪能量攒起来。这样不会打断音乐 App 的主体验，而是让播放过程多一层陪伴反馈。

产品游戏拓展可以定义为“情绪能量补给站”。用户选择情绪后，小宠物进入对应状态；播放歌曲时，系统根据播放时长、节奏和当前情绪持续积累情绪能量。以 EMO 状态为例，小宠物缩在雨夜角落里，歌曲播放时掉落“歌词碎片”“雨滴音符”“共鸣光点”，用户可以帮它收集这些光点，为它加油。能量满后，小宠物不会立刻“变开心”，而是从“低电量 EMO”变成“被陪伴的 EMO”。重点不是消灭负面情绪，而是承认情绪，陪它待一会儿。

为什么要加入游戏化

  • 情绪设置解决“我现在是什么感觉”，小游戏进一步解决“我怎么陪这个情绪待一会儿”。
  • 播放页原本是工具型空间，宠物和能量条能把听歌过程变成有反馈的陪伴场景。
  • 掉落、收集、解锁动作和装饰，可以自然承接留存、会员权益、皮肤套装和活动运营。

首个玩法：雨夜补给站

  • 用户选择 EMO 或深夜 EMO 后，播放歌曲时进入雨夜场景。
  • 场景上方掉落雨滴音符、歌词碎片、共鸣光点，点击或接住后增加共鸣能量。
  • 能量满后，小宠物从“低电量 EMO”变成“被陪伴的 EMO”，雨变小，小夜灯亮起。

玩法边界

  • 不设置失败、不扣分、不 Game Over，避免给低落情绪用户增加压力。
  • 听歌自动积累能量，用户不操作也能推进；互动只作为加速和情绪表达。
  • 奖励优先做小动作、房间物件、皮肤套装和今日情绪卡，不抢推荐和播放主线。

后续所有情绪可以复用同一套底层机制：播放触发、掉落物、能量条和宠物反馈。差异主要体现在场景主题、掉落物和奖励表达。

Tab 4 · MVP

用 demo 跑通“选情绪、看团子、听推荐”的最小闭环。

MVP 以现有 FloatingBeatMascot 为基础，完成播放页浮动团子、节奏动效、一级情绪选择、二级动作状态、情绪推荐位和推荐解释，让用户直接感知“情绪定基调，动作状态定场景”。

Demo 效果

页面中出现一个浮动情绪团子。播放歌曲后，团子跟随节奏上下弹动、倾斜、发光；用户点击团子可切换基础形象和一级情绪；点击“正在旅行”可叠加二级动作状态皮肤。

放松团子正在匹配下一首

降低节奏强度，减少强鼓点，保留温柔人声。

MVP 上线内容

  • 一级情绪：开放 12 种一级情绪入口，并在 demo 中优先展示平静、快乐、放松、EMO、元气等高频情绪。
  • 二级动作状态：开放正在旅行/旅行家、正在通勤/驾驶、正在健身、正在助眠、深夜 EMO、海边放松。
  • 播放页陪伴：团子以浮动层出现在播放页，支持拖拽、点击切换和动作状态皮肤切换。
  • 节奏反馈：根据真实音频频谱或播放进度驱动弹跳、倾斜、缩放和发光。
  • 推荐联动：展示“团子为你推荐”的情绪歌单，并用一句话说明推荐原因。
体验目标 用户完成“选择一级情绪 - 切换二级动作状态 - 播放歌曲 - 看到团子反馈 - 点击推荐”的完整路径。
验证指标 推荐点击率、播放页停留、情绪切换率、分享率。
前端框架 当前项目基于 React 18、Vite、Tailwind CSS、Redux Toolkit、React Router。MVP 可直接在现有 lyriks 原型中迭代。
播放状态接入 通过 Redux playerSlice 读取 activeSong 和 isPlaying，通过 AudioPlayerContext 获取 audioElement。
音频频谱驱动 FloatingBeatMascot 使用 AudioContext、createMediaElementSource、AnalyserNode，读取低频、中频、全频和时域能量。
动效策略 根据 beatLevel 计算 bounce、glow、tilt，并按不同团子的 motion profile 调整弹跳高度、缩放、倾斜和响应速度。
交互能力 Pointer Events 支持拖拽，点击切换团子形象，按钮切换“正在旅行”动作状态皮肤。位置会在窗口缩放时自动 clamp 到视口内。
降级方案 当真实频谱不可用或持续静音时，使用 currentTime 的 fallback beat，保证 demo 在跨域音频或浏览器限制下仍有可见节奏。

Tab 5 · 商业闭环

从推荐增强到会员权益，再到虚拟装扮和品牌联名。

收益逻辑来自《收益闭环.docx》：情绪团子不是趣味贴片，而是把听歌、推荐、会员、社交和轻量游戏奖励包装成一个更有情绪感和可消费空间的虚拟形象系统。

第一层：提升听歌与推荐消费

  • 团子让播放页从静态工具变成有反馈、有陪伴、有互动的空间。
  • “兴奋团子为你推荐”“放松团子正在匹配”比普通推荐更容易被理解和点击。
  • 核心指标：播放页停留、推荐点击率、播放转化、完播率。

第二层：增加会员付费感知

  • 基础团子免费，会员解锁更多团子形象、动态表现、二级动作状态、专属音效和歌词背景。
  • 会员权益从“无广告、高音质、下载”扩展到年轻用户更容易感知的形象表达。
  • 核心指标：会员入口点击、皮肤试用、会员转化率。

第三层：虚拟道具与联名收入

  • 状态皮肤套装、动作表现、背景场景、表情包、音效包可以成为可购买或可解锁资产。
  • 歌手新专辑、演唱会、影视 OST 和品牌活动可以推出限定团子形象或状态套装。
  • 核心指标：道具购买、活动参与、分享曝光、联名转化。

最终建议：情绪团子的 MVP 要先把“音乐推荐的前台角色”跑通，再逐步升级为“用户音乐身份资产”。主线始终围绕听歌、推荐、会员和分享，不让宠物玩法脱离音乐 App 主流程。
