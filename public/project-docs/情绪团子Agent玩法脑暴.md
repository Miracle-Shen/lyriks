# 情绪团子 Agent 玩法脑暴

基于《产品方案》中已有的“一级情绪 + 二级动作状态 + 音乐推荐 + 陪伴宠物”框架，情绪团子后续可以从播放页陪伴形象升级为音乐 Agent。它不只是改变外观或推荐歌曲，而是能理解用户当下状态、参与听歌过程、组织音乐生活，并在社交关系中承担情绪表达角色。

## 一、Agent 核心定位

情绪团子 Agent 可以被定义为“音乐 App 内的情绪陪伴型音乐伙伴”，主要具备三层能力：

### 1. 情绪感知者

Agent 根据用户主动选择的一级情绪、二级动作状态，以及听歌行为进行判断，包括：

- 当前选择的情绪状态
- 当前选择的动作场景
- 歌曲跳过、收藏、循环、分享等行为
- 听歌时间段
- 歌词氛围
- 最近常听曲风与歌单偏好

它的目标不是诊断用户情绪，而是更好地理解用户此刻想用什么方式听歌。

### 2. 陪听反应者

Agent 在播放页或音乐空间中实时陪伴用户听歌，并根据音乐节奏、歌词氛围、音量起伏和情绪转折做出反应。

例如：

- 前奏时进入场景
- 副歌时动作增强
- 慢歌时轻晃、漂浮或缩成一团
- 高能歌曲时跳跃、发光或跑动
- 歌曲结束后给出一句轻量反馈

这能把“播放音乐”转化为“有人陪我一起听这首歌”的体验。

### 3. 音乐行动者

Agent 不只是被动展示，还可以主动帮助用户完成音乐行为：

- 生成情绪歌单
- 调整播放队列
- 发起陪伴任务
- 整理情绪听歌记录
- 推荐适合当前状态的音乐内容
- 参与好友之间的音乐互动

## 二、可落地 Agent 玩法

### 1. 今日情绪接管

用户打开 App 后，团子主动询问：

> 今天想让哪种状态陪你？

如果用户不主动选择，Agent 可以根据最近听歌行为和时间段给出建议：

> 我猜你现在可能想放松，要不要进入海边放松模式？

这个玩法可以强化“团子理解我”的感觉，同时降低用户选择状态的成本。

### 2. 一键生成情绪歌单

用户选择“一级情绪 + 二级动作状态”后，Agent 不只是推荐已有歌单，而是生成一组更贴合当下的播放队列。

例如选择“放松 + 海边放松”时，Agent 可以生成：

- 前 3 首用于降低节奏，帮助用户进入状态
- 中段保持 Chill、轻松、海边氛围
- 结尾使用一首温暖或舒缓的歌曲收尾

相比普通推荐，这更像“团子为我安排了一段听歌旅程”。

### 3. 陪听小剧场

团子在一首歌内根据音乐结构做出不同表现：

- 前奏：进入场景，准备陪听
- 主歌：保持轻动作，稳定氛围
- 副歌：动作增强，表情更明显
- Bridge 或情绪转折：出现特殊动作
- 尾奏：慢慢收回动作，给出一句反馈

不同状态可以拥有不同小剧场表现：

- 深夜 EMO：雨滴、暗光、缩成一团
- 海边放松：海浪、晒太阳、轻晃
- 正在健身：跑动、跳跃、能量光效
- 正在旅行：拿相机、看地图、张望

### 4. 情绪续航模式

Agent 根据用户的连续听歌状态，判断是否需要延续、转场或轻微拉回。

例如用户连续听深夜 EMO 40 分钟后，团子可以提示：

> 要不要我帮你把最后三首慢慢换成治愈一点的？

这个玩法要保持克制，所有情绪转向都应该由用户确认，避免让用户觉得被干预。

### 5. 状态任务

每个二级动作状态都可以设计轻量任务，用来解锁动作、道具或场景变化。

示例：

- 旅行家：听 5 首带城市、海边、路途感的歌曲，解锁相机动作
- 助眠：连续 3 晚使用助眠模式，解锁月光被子
- 健身：完成 30 分钟高能播放，解锁能量光效
- 通勤：连续 5 次在通勤时间听歌，解锁车窗背景
- 深夜 EMO：收藏 3 首深夜歌曲，解锁雨滴场景

任务不应过度刷数，而是服务真实听歌场景。

### 6. 团子日记

Agent 根据用户每日听歌状态生成简短音乐情绪记录。

例如：

> 你今天在 22:40 进入了低落状态，最后停在一首比较温柔的歌上。

也可以生成周报内容：

- 本周最常出现的情绪
- 本周最常使用的动作状态
- 本周代表歌曲
- 本周情绪转场路径
- 本周团子解锁的新动作

这个玩法可以沉淀用户和团子之间的长期关系。

### 7. 情绪转场歌单

用户可以直接表达：

> 我不想一直 EMO，帮我慢慢转出去。

Agent 生成一组情绪转场队列，例如：

EMO -> 低落 -> 治愈 -> 平静

这种玩法比单纯推荐“治愈歌单”更细腻，因为它承认用户当前状态，并提供一个温和过渡。

## 三、社交 Agent 玩法

### 1. 好友团子来访

好友展示的不只是头像，而是带有情绪和动作状态的音乐形象。

例如：

- 好友 A：元气 + 正在通勤
- 好友 B：放松 + 海边放松
- 好友 C：EMO + 深夜 EMO

好友团子可以进入用户音乐空间短暂停留，形成更强的音乐社交感。

### 2. 好友风格分身

Agent 可以读取好友公开歌单或授权歌单，生成一个临时的“好友风格团子”。

这个团子不是复制好友本人，而是把好友的音乐口味转化成一个陪听形象。

示例：

> 这是小林的音乐风格团子，它更喜欢 R&B、夜晚、低亮度的歌。

### 3. 双人情绪混音

两个用户选择各自状态后，Agent 生成一份折中歌单。

例如：

- 用户 A：放松 + 海边放松
- 用户 B：浪漫 + 小酒馆
- 生成结果：海边小酒馆、晚风、R&B、轻情歌

这个玩法适合好友、情侣、搭子一起听歌。

### 4. 给好友投喂一首歌

用户可以让自己的团子带着一首歌去好友空间。

示例：

> 我的团子想送你一首今天适合你的歌。

相比普通歌曲分享，这种方式更有情绪表达感，也更符合“陪伴宠物”的产品设定。

## 四、更有想象力的玩法

### 1. 团子 DJ

用户只需要给出一句模糊指令：

> 今晚别让我太丧。

团子 DJ 自动负责控场，包括：

- 选择起始歌曲
- 调整整体情绪走向
- 控制节奏变化
- 生成下一首推荐理由
- 在合适时机切换状态

它的价值是把“找歌”变成“被照顾地听歌”。

### 2. 音乐情绪地图

Agent 把用户在不同时间、地点、状态下听过的歌沉淀成一张情绪地图。

例如：

- 雨天常听区域
- 深夜常回到的歌
- 旅行时反复播放的歌曲
- 通勤时最常出现的情绪

旅行家状态可以和音乐情绪地图强绑定，形成“团子带着我回到某段音乐记忆”的体验。

### 3. 歌词共情 Agent

当歌词出现强情绪片段时，团子可以做轻量反馈。

例如：

> 这句好像戳到你了。

用户也可以把歌词片段收藏成“情绪句子”，之后由 Agent 生成回忆卡或情绪日记。

### 4. 团子性格成长

不同用户的团子可以逐渐形成不同陪伴风格。

示例：

- 安静型：少说话，多用动作陪伴
- 活力型：更常鼓励用户，主动推荐高能歌曲
- 细腻型：更关注歌词、情绪和听歌转折
- DJ 型：更擅长控场和组织播放队列

这样团子会从“功能”逐渐变成“关系”。

### 5. 状态盲盒

Agent 根据用户最近听歌行为，解锁隐藏动作状态。

示例：

- 雨天散步
- 凌晨便利店
- 独自看海
- 周末大扫除
- 考试冲刺
- 小酒馆微醺

这些状态不是随机皮肤，而是由真实听歌行为触发，因此更容易形成情绪记忆点。

## 五、MVP 后优先级建议

### 第一阶段：建立 Agent 基础感

优先实现：

- 情绪和动作状态选择
- Agent 生成播放队列
- 播放页陪听动作
- 简单情绪日记

目标是验证用户是否能理解“团子不仅是皮肤，而是会陪听和推荐的音乐伙伴”。

### 第二阶段：强化陪伴与成长

继续实现：

- 情绪转场歌单
- 状态任务成长
- 团子性格偏好
- 更多动作状态解锁

目标是让用户愿意长期使用，并形成“我的团子越来越懂我”的感受。

### 第三阶段：进入社交表达

再扩展：

- 好友团子来访
- 双人情绪混音
- 好友风格分身
- 团子投喂歌曲

目标是让情绪团子成为音乐社交中的新表达载体。

## 六、最值得优先打磨的两个方向

### 1. 情绪转场歌单

这是最有功能价值的 Agent 能力。它不是简单地推荐某类歌，而是帮助用户从一种情绪慢慢过渡到另一种情绪。

它能体现情绪团子的理解能力，也能和音乐推荐能力形成强绑定。

### 2. 陪听小剧场

这是最容易建立产品记忆点的表现层能力。用户一进入播放页，就能直观看到团子和音乐产生关系。

它能强化“有人陪我一起听”的感受，是情绪团子区别于普通音乐推荐功能的关键体验。
