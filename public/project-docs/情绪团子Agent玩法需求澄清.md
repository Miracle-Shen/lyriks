# 情绪团子 Agent 玩法需求澄清

> 基于《情绪团子 Agent 玩法脑暴》整理。本文用于在 `lyriks` 开发前确认产品边界、交互流程、实现范围和验收标准。

## 1. 背景

情绪团子当前已经具备“播放时浮动陪伴形象 + 情绪状态 + 动作状态 + 皮肤/特效”的基础能力。后续希望把团子从单纯的播放页陪伴形象，升级为更主动的音乐情绪 Agent。

本轮优先落地两个方向：

1. 用户进入首页，也就是发现页时，团子可以主动发起“今日情绪接管”。
2. “陪听小剧场”不单独做成新页面、剧场页或独立模式，而是收敛为团子主体形象附近随机生成的临时特效，特效会自动消失。

## 2. 本轮目标

### 2.1 今日情绪接管

用户进入发现页后，团子以轻量方式主动询问今天想进入什么情绪/听歌状态，帮助用户更快完成“一级情绪 + 二级动作状态”的选择，并给后续推荐、播放和陪伴表现提供上下文。

核心感受：

- 团子主动但不打扰。
- 用户不用先进入播放页，也能感受到团子的 Agent 属性。
- 用户可以一键接受推荐，也可以手动选择情绪和动作状态。
- 用户拒绝或关闭后，不应频繁再次打扰。

### 2.2 陪听小剧场轻量化

陪听小剧场不建设成独立页面，也不做复杂剧情系统。它只作为播放过程中围绕团子主体形象出现的临时视觉反馈，用随机小特效表达“团子正在陪我听这首歌”。

核心感受：

- 特效围绕团子出现，不抢主流程。
- 特效与当前情绪、动作状态、播放节奏存在弱关联。
- 特效短暂出现后自动消失。
- 不新增“剧场”入口，不改变当前路由结构。

## 3. 范围定义

### 3.1 本轮包含

- 在发现页触发今日情绪接管入口。
- 支持推荐一个默认情绪/动作组合。
- 支持用户接受推荐、手动切换或关闭。
- 将今日选择同步到团子当前情绪与动作状态。
- 在团子主体形象附近随机生成临时陪听特效。
- 特效自动消失，不需要用户手动关闭。
- 保留现有播放页/全局浮动团子能力。

### 3.2 本轮不包含

- 不新增独立“陪听小剧场”页面。
- 不新增复杂剧情、分镜、台词系统。
- 不做真实 AI 情绪诊断。
- 不接入新的后端、账号系统或云端记忆。
- 不实现完整歌单生成引擎，只保留为后续能力预留接口/文案。
- 不做社交 Agent、好友团子、团子日记、情绪地图等长期玩法。

## 4. 页面与入口

### 4.1 触发页面

触发位置：发现页，也就是当前 `Discover` 页面。

触发时机：

- 用户进入 `/` 路由。
- 页面基础内容已渲染后再出现，不阻塞歌曲列表加载。
- 每个自然日最多主动出现一次。
- 如果用户关闭，当天不再主动弹出。
- 如果用户已经在当天完成选择，当天不再主动弹出。

### 4.2 表现方式

建议采用轻量浮层或团子气泡，而不是全屏弹窗。

推荐形态：

- 靠近团子主体位置出现一个短气泡。
- 气泡内包含一句主动询问文案。
- 提供 2 到 3 个主要操作：
  - 接受推荐
  - 换一个状态
  - 今天先不用/关闭

示例文案：

- “今天想让我用哪种状态陪你？”
- “我猜你现在适合放松一点，要进入海边放松吗？”
- “今天让我接管一下听歌氛围？”

## 5. 今日情绪接管流程

### 5.1 默认推荐逻辑

本轮可以先使用本地规则，不需要真实模型：

- 上午：元气 + 正在通勤
- 下午：专注 + 正在通勤
- 晚上：放松 + 海边放松
- 深夜：EMO/低落 + 深夜 EMO
- 如果当前已有用户选择，则优先沿用上次选择作为备选。
- 如果推荐组合不满足动作状态的情绪约束，则自动选择该情绪下的第一个可用动作状态。

### 5.2 用户操作

接受推荐：

- 更新团子当前一级情绪。
- 更新团子当前二级动作状态。
- 关闭今日情绪接管气泡。
- 记录当天已完成接管。

换一个状态：

- 打开现有团子设置能力，或在气泡内展示简化选择器。
- 用户选择后同步到团子状态。
- 记录当天已完成接管。

关闭/今天先不用：

- 关闭气泡。
- 记录当天已关闭。
- 当天不再主动弹出。

### 5.3 异常与边界

- 如果当前没有播放歌曲，今日情绪接管仍然可以出现。
- 如果当前团子因为“无播放歌曲”被隐藏，需要允许发现页的今日情绪接管拥有自己的入口表现，或让团子在首页以静态状态出现。
- 如果用户正在拖动团子，不弹出/不更新气泡位置，避免误触。
- 如果本地存储不可用，功能仍可展示，只是不保证当天只出现一次。

## 6. 陪听小剧场特效

### 6.1 产品定义

陪听小剧场 = 播放过程中，围绕团子主体形象随机生成并自动消失的情绪特效。

它不是：

- 新页面
- 新路由
- 剧场模式
- 弹窗剧情
- 可交互小游戏

### 6.2 触发条件

基础触发：

- 有当前播放歌曲。
- 播放状态为 `isPlaying = true`。
- 团子主体形象可见。

触发节奏：

- 每隔一段随机时间生成一次，例如 6 到 14 秒。
- 每次只出现 1 到 3 个小特效。
- 特效持续 1.2 到 2.8 秒后自动消失。
- 页面不可见、暂停播放或无歌曲时停止生成。

### 6.3 特效位置

特效必须围绕团子主体形象生成：

- 出现在团子外圈、头顶、左右侧或脚边。
- 不遮挡播放器主要按钮。
- 不遮挡发现页歌曲列表中的歌曲卡片核心信息。
- 不影响团子拖拽和设置入口点击。

### 6.4 特效类型

特效可以基于当前一级情绪和二级动作状态选择：

| 状态 | 可用特效 |
| --- | --- |
| 深夜 EMO | 雨滴、暗色微光、低频波纹 |
| 海边放松 | 小浪花、阳光点、轻风线条 |
| 正在健身 | 能量粒子、跳动光点、速度线 |
| 正在旅行 | 小地图点、相机闪光、路线路径 |
| 正在助眠 | 月光点、星星、柔和呼吸环 |
| 正在通勤 | 城市窗光、流动线、节奏点 |

本轮不要求全部实现，可以先完成 3 到 4 类，并保证后续易扩展。

### 6.5 行为要求

- 特效无需用户操作。
- 特效自动淡出并从 DOM/状态中移除。
- 多个特效不能无限累积。
- 团子拖拽时，特效应跟随团子容器或暂停生成。
- 打开团子设置面板时，特效可以减少或暂停，避免视觉拥挤。

## 7. 状态与数据

### 7.1 建议状态

需要在团子功能层管理：

- 当前一级情绪 `emotionId`
- 当前二级动作状态 `actionId`
- 今日情绪接管展示状态
- 今日是否已接受/关闭
- 当前陪听临时特效队列

### 7.2 本地持久化

建议使用 `localStorage` 保存：

- 最近一次选择的 `emotionId`
- 最近一次选择的 `actionId`
- 今日接管记录日期
- 今日接管结果：accepted / dismissed

日期按用户本地时间自然日计算即可。

### 7.3 未来预留

未来如果接入真实 Agent 或推荐服务，可以把今日推荐来源从本地规则替换为：

- 最近常听歌曲
- 最近常选情绪
- 当前时间段
- 跳过/收藏/循环行为
- 歌词和曲风标签

本轮文案应避免承诺“我已准确判断你的情绪”，建议使用“我猜你可能想……”这类弱判断表达。

## 8. 交互与视觉原则

- 主动出现，但必须能一键关闭。
- 气泡和特效的视觉层级低于核心播放控制。
- 文案轻、短、像陪伴，不像系统通知。
- 不使用医疗化、诊断化措辞。
- 不强迫用户进入某种情绪，也不把负面情绪导向评价为错误。
- 特效应服务团子陪伴感，不应变成页面背景噪声。

## 9. 验收标准

### 9.1 今日情绪接管

- 进入发现页后能看到今日情绪接管提示。
- 接受推荐后，团子的情绪和动作状态发生对应变化。
- 关闭后，当天再次进入发现页不再主动弹出。
- 接受推荐后，当天再次进入发现页不再主动弹出。
- 不影响发现页歌曲列表加载、滚动和流派选择。
- 无播放歌曲时，用户仍能完成今日状态选择。

### 9.2 陪听小剧场特效

- 播放歌曲时，团子附近会随机出现临时特效。
- 暂停播放后，不继续生成新特效。
- 特效会自动淡出并消失。
- 特效不会遮挡播放器核心操作。
- 不存在特效无限累积造成页面卡顿的问题。
- 不新增“陪听小剧场”页面、路由或导航入口。

## 10. 待确认问题

1. 首页无播放歌曲时，是否允许团子以静态状态常驻显示，专门承载今日情绪接管？
2. 今日情绪接管的“换一个状态”是复用现有团子设置面板，还是做一个更轻量的首页选择器？
3. 今日情绪选择是否要立即影响发现页歌曲推荐，还是本轮只影响团子表现？
4. 本轮是否需要把选择结果展示在首页标题区域，例如“今日状态：放松 · 海边”？
5. 陪听特效的首批状态优先级是否确定为：深夜 EMO、海边放松、健身、助眠？
6. 是否需要提供一个“关闭动态特效”的设置项，照顾偏安静的用户？

## 11. 建议开发拆分

### 阶段一：状态与首页接管

- 抽出团子情绪/动作状态管理，避免只存在于浮动组件内部。
- 在发现页增加今日情绪接管气泡。
- 接入本地日期记录，控制每日触发频率。
- 完成接受、关闭、换状态基础流程。

### 阶段二：团子附近特效

- 新增轻量特效组件，挂载在团子主体容器附近。
- 根据情绪/动作状态选择特效池。
- 根据播放状态定时生成和清理特效。
- 处理拖拽、暂停、设置面板打开等边界。

### 阶段三：体验打磨

- 调整动画强度、出现频率和移动端位置。
- 检查播放器、歌曲列表、气泡之间的遮挡关系。
- 增加本地存储异常兜底。
- 补齐核心交互的手动验证清单。
